How should the app be built?

Three approaches — each has real trade-offs for your laptop + iPad setup.

A

Frontend Only (No Backend)

Pure HTML/CSS/JavaScript. Data lives in your browser. Works offline instantly, easiest to build.

Pros

  • Simplest to build
  • No server needed
  • Works offline

Cons

  • Notes won't sync between laptop and iPad
  • Data lost if you clear browser
B

Full-Stack (Node.js + SQLite) ⭐ Recommended

A small server runs on your laptop. iPad and laptop both connect to it on your home network.

Pros

  • Notes sync between devices
  • Data stored properly in a database
  • Full control, no third-party

Cons

  • Server must be running on your laptop to access from iPad
  • Slightly more to build
C

Frontend + Supabase (Cloud Database)

Frontend app connected to a free cloud database. Works anywhere, anytime from any device.

Pros

  • Syncs everywhere, always
  • No server to manage
  • Works away from home

Cons

  • Depends on a third-party service
  • Requires internet connection
